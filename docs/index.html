XZLANG Documentation

XZLANG is a statically typed systems language that compiles to x86_64 assembly via NASM. No virtual machine, no garbage collector, no runtime.

Native compilation

Compiles directly to x86_64 instructions. Output is a stripped ELF binary.

:

Type syntax

Types precede names, separated by a colon: i64:x.

No runtime

No garbage collector. No VM. Memory management is explicit.

Start here: See the Variables section for a quick overview of the type system, then move to Functions.

Variables

Variables are declared with a colon separating the type from the name. Every statement must end with .

xzlang · variable declarations 
i64:age = 25.       ; 64-bit integer
str:msg = "Hello".    ; string
f64:pi  = 3.14.      ; 64-bit float
bool:ok = true.      ; boolean

Functions

Parameters are passed in curly braces {}. The function body is enclosed in parentheses ().

xzlang · function definition and call 
i64 calculate{i64:a, i64:b}(
    ret a + b * 2.
)

i64 start{ntg}(
    i64:res = calculate{10, 5}.
    prms{"Result: ~i", ~res}.
    ret 0.
)
start is the program entry point, equivalent to main in C. The ntg parameter means "no type guard" — the argument is untyped.

Control Flow

If / Else

Conditions go inside {}. The block body goes inside ().

xzlang · conditionals 
if{x == 10}(
    prms{"ten"}.
) else (
    prms{"not ten"}.
)

Loops

XZLANG supports for and while. Arguments in for follow the standard init, condition, post pattern.

xzlang · for loop 
for{i64:i = 0, i < 5, ++i}(
    prms{"step ~i", ~i}.
)

Ternary

Syntax: condition => if_true | if_false.

xzlang · ternary operator 
i64:val = 10 > 5 => 999 | 111.
prms{"Result: ~i", ~val}.

Arrays

Fixed-size arrays. Elements are zero-indexed.

xzlang · arrays 
; initializer list
i64:arr = [10, 20, 30].
prms{"arr[1]: ~i", ~arr[1]}.

; fixed-size uninitialized array
i64:arr2[5].
arr2[0] = 777.
Out-of-bounds access is a compile-time error when the array size is statically known.

Pointers

Pointers give direct access to memory addresses.

SyntaxMeaning
addr:xTake the address of variable x
^pDereference pointer p
~pPrint address via prms
xzlang · pointers 
i64:val = 100.
ptr<i64>:p = addr:val.

prms{"address: ~p", ~p}.
prms{"value:   ~i", ^p}.

Heap allocation

Use gain: to allocate and drop: to free. There is no garbage collector — memory leaks are the programmer's responsibility.

xzlang · heap allocation 
ptr<i64>:p = gain:i64[3].
p[0] = 123.

drop:p.

Libraries

Libraries are imported with the using keyword. Standard libraries are located in lib/.

Multiple libraries can be imported in a single statement:

xzlang · imports 
using{"tml", "net"}.  ; multiple at once

; or separately:
using{"tml"}.
using{"net"}.

Standard libraries

LibraryDescriptionFunctions
tml Basic I/O prms, get
net TCP/IP networking connect, listen, send

CLI

The compiler accepts the following commands and flags. By default, build runs strip on the output binary.

shell · commands 
# compile to binary
./xzlang build main.xzl -o my_app

# run via interpreter
./xzlang run main.xzl

# print generated assembly
./xzlang build main.xzl --emit-asm

Flags

FlagDescription
-o <name>Output binary name
--emit-asmPrint generated x86_64 assembly
--no-stripKeep debug symbols in the output
--verboseVerbose compilation log
